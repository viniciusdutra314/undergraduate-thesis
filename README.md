# Graph Traffic TCC — Julia + Typst pipeline

This repository uses **Julia** to generate simulation data and plots, then **Typst** to typeset the thesis and slides using those outputs. The heavy simulations are executed by a Rust CLI (`graph_traffic_simulator`), which the Julia code installs automatically.

## Project layout

- `src/`
  - `GraphTraffic.jl`: shared utilities (schema, engine, topology helpers, analysis helpers).
  - `rho_vs_atraso.jl`: generates data and plots for the main “rho vs atraso” experiment.
  - `watts_messages_vs_time.jl`: generates data and plots for Watts–Strogatz message counts over time.
  - `watts_classical_plot.jl`: produces the classic Watts–Strogatz L/C vs beta plot (no simulation data).
- `raw_results/`: HDF5 simulation outputs produced by the Rust CLI.
- `thesis/assets/plots/`: SVG plots generated by Julia.
- `thesis/assets/tables/`: CSV tables generated by Julia.
- `thesis/main.typ`: thesis document that references assets in `thesis/assets/`.
- `thesis/slides.typ`: slides that import definitions from `main.typ`.

## Requirements

- Julia (uses the environment in `Project.toml` + `Manifest.toml`).
- Rust toolchain (`cargo`) — required because the Julia scripts call `cargo install` for the simulator.
- Typst **0.14.2** (the Typst files assert this version).

## Setup (Julia environment)

From the repository root:

- `julia --project=@. -e "using Pkg; Pkg.instantiate()"`

This installs all Julia dependencies defined in `Project.toml`.

## How the Julia scripts work

### `src/GraphTraffic.jl`

This file defines the shared infrastructure used by all experiments:

- **`GraphTraffic.Schema`**: data structures for simulator configuration (routing method, observers, modifiers, etc.).
- **`GraphTraffic.Engine`**:
  - `run_rust_cli(...)` installs and runs the Rust simulator (`graph_traffic_simulator`) and writes `.hdf5` outputs to `raw_results/`.
  - `open_raw_results(...)` opens HDF5 outputs by name.
  - `make_cli(...)` builds simple CLI entrypoints (`simulate`, `plot`, `all`).
- **`GraphTraffic.Topology`**: graph generators, connectivity enforcement, and graph metrics.
- **`GraphTraffic.SharedData`**: common parameters + lazy graph builders for the main topologies.
- **`GraphTraffic.Analysis`**:
  - computes metrics (delay, traveling time, free-flow rate).
  - `save_dataframe_to_csv(...)` writes to `thesis/assets/tables/`.
  - `save_figure(...)` writes to `thesis/assets/plots/`.

### `src/rho_vs_atraso.jl` (`RhoVsAtraso`)

- **`generate_data()`**:
  - Builds comparable graphs for the four topologies.
  - Writes `thesis/assets/tables/graph_stats.csv`.
  - Runs the Rust simulator, producing `raw_results/varying_message_generation.hdf5`.
  - Augments the HDF5 file with edge betweenness and free-flow rate datasets.
- **`plot()`**:
  - Produces:
    - `thesis/assets/plots/p_critico_delay.svg`
    - `thesis/assets/plots/p_critico_travel.svg`
    - `thesis/assets/plots/p_critico_free_flow.svg`
    - `thesis/assets/plots/p_critico_betweeness.svg`
  - Writes `thesis/assets/tables/betweeness_vs_messages.csv`.

### `src/watts_messages_vs_time.jl` (`WattsStrogatzBetaVsLAndC`)

- **`generate_data()`**: runs simulations for a set of Watts–Strogatz graphs with varying beta and writes `raw_results/watts_strogatz_varying_beta.hdf5`.
- **`plot()`**: produces `thesis/assets/plots/watts_messages_vs_time.svg`.

### `src/watts_classical_plot.jl` (`WattsStrogatzClassicPlot`)

- **`plot()`**: generates `thesis/assets/plots/watts_classical_plot.svg` directly from graph statistics (no simulation data).

### `recreate_all_tcc.jl`

A convenience launcher that runs all experiments in a single command:

- `julia --project=@. recreate_all_tcc.jl simulate_all`
- `julia --project=@. recreate_all_tcc.jl plot_all`
- `julia --project=@. recreate_all_tcc.jl simulate_and_plot_all`

## Running individual scripts

Each experiment module uses the same CLI contract provided by `make_cli(...)`:

- `julia --project=@. src/rho_vs_atraso.jl simulate`
- `julia --project=@. src/rho_vs_atraso.jl plot`
- `julia --project=@. src/rho_vs_atraso.jl all`

(Replace the file path with `src/watts_messages_vs_time.jl` or `src/watts_classical_plot.jl` as needed.)

## Typst integration

The Typst documents read the generated outputs from `thesis/assets/`:

- `thesis/main.typ` uses `image("assets/plots/*.svg")` and `csv("assets/tables/*.csv")` for figures and tables.
- `thesis/slides.typ` imports definitions from `main.typ` and can reuse the same assets.

After generating the plots/tables, compile Typst from the repo root:

- `typst compile thesis/main.typ`
- `typst compile thesis/slides.typ`

## Notes

- If you run `plot` before running `simulate`, the HDF5 files in `raw_results/` must already exist.
- The Rust simulator is installed automatically the first time `run_rust_cli(...)` runs:
  - `cargo install --git https://github.com/viniciusdutra314/GraphTraffic-rs --rev ed127dcdd65aeeca5f25765a695a08a6dcfd21cf --locked`
  - Ensure `cargo` is available in your PATH.
